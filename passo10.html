Este passo é sobre a execução da sua lógica de negócio sem se preocupar com servidores. Você se concentra apenas no código.

1. AWS Lambda

O que é: Computação serverless. Você escreve uma função em uma linguagem (Python, Node.js, Java, etc.) e a AWS a executa sob demanda, escalando automaticamente e você paga apenas pelo tempo de computação usado.

Analogia: Um chef sob demanda. Você não contrata um chef em tempo integral. Você só chama ele (e paga) quando tem um prato para ser preparado (um evento para processar). Ele some quando termina.

Como se integra: A Lambda é a consumidora dos serviços do Passo 9. Ela é acionada por mensagens de uma fila SQS, por notificações de um tópico SNS, por regras do EventBridge, e muitas outras coisas.

2. API Gateway

O que é: Um serviço para criar, publicar e gerenciar APIs RESTful e WebSocket.

Analogia: O porteiro de um prédio comercial. Ele recebe todas as visitas (requisições HTTP), verifica se elas têm autorização (autenticação), descobre para qual escritório (Lambda function) a pessoa quer ir e a direciona.

Como se integra: O API Gateway é a porta de entrada para suas Lambdas a partir do mundo exterior (web, mobile apps). Ele recebe uma requisição HTTP e a transforma em um evento para disparar uma função Lambda.
