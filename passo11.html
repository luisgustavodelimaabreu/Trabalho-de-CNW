Este passo é sobre visibilidade. Em um ambiente complexo e sem servidores, você não pode "entrar na máquina" para debugar. Você precisa de ferramentas para observar o comportamento da aplicação.

1. CloudWatch

O que é: O serviço central de monitoramento e observação da AWS.

Logs: Toda função Lambda, todo acesso ao API Gateway, e muitos outros serviços, geram "logs" (registros de texto) que são armazenados no CloudWatch Logs. É seu primeiro lugar para buscar quando algo dá errado.

Métricas: Números que representam o desempenho do seu sistema. Ex: Quantas vezes sua Lambda foi executada? Quantos errors ocorreram? Qual a latência do API Gateway? Você pode criar dashboards e alarmes com base nessas métricas.

Analogia: A caixa preta e os painéis de instrumentos de um avião.

2. AWS X-Ray

O que é: Um serviço de rastreamento distribuído. Ele ajuda você a visualizar a jornada completa de uma requisição à medida que ela percorre diversos serviços da sua arquitetura (API Gateway -> Lambda -> SQS -> outra Lambda -> DynamoDB).

Objetivo: Identificar gargalos de performance (onde a requisição está travando ou lentidão) e entender as dependências entre os serviços. Ele desenha um mapa (um trace) de como os serviços interagem para atender uma solicitação.

Analogia: Um rastreador GPS para suas requisições. Você vê exatamente qual caminho ela percorreu e quanto tempo gastou em cada ponto.
